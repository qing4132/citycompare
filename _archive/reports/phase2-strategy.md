## 1. 现状审计

### 1.1 项目概况

| 维度 | 现状 |
|------|------|
| 产品 | 全球城市对比工具，154 城市 × 26 职业 × 81 国税制 |
| 年龄 | 19 天（2026-03-23 至今），294 次提交 |
| 技术栈 | Next.js 15 + TypeScript + Tailwind + Recharts |
| 团队 | 1 人 + AI 辅助 |
| 域名 | whichcity.run（已上线） |
| 语言 | zh/en/ja/es |
| 流量 | 未知（有 GA4 但无数据分享） |
| 营收 | 零 |

### 1.2 19 天做到的事（令人印象深刻）

- 从 0 到 154 城市完整数据集
- 81 国税制引擎（含外派方案）
- 4 语言国际化（1960 行翻译文件）
- 数字游民板块（签证、VPN、免签矩阵）
- 自动化汇率更新 + CI
- 单元测试覆盖核心算法
- 深色模式、SEO、OG 图片
- 档案化的数据审计流程

**这不像一个 19 天的项目。数据密度和工程完成度很高。**

### 1.3 然而——

这个项目目前处于一个典型的工程师驱动产品的陷阱：

**特征越多，价值越模糊。**

154 城市、26 职业、50+ 数据维度、22 个排行榜 Tab、数字游民签证、4 本护照免签、4 个时区重叠——任何单独拿出来都有价值，组合在一起却产生了信息过载。

用户打开页面的第一感受不是 "这正是我需要的"，而是 "这也太多了"。

## 2. 核心问题诊断

### 2.1 定位模糊

当前的定位摇摆在三个身份之间：

| 身份 | 表现 | 问题 |
|------|------|------|
| **城市数据库** | 50+ 字段、子指标全展开、排行榜 22 Tab | 没人需要"城市百科" |
| **移居决策工具** | 收入/成本/储蓄的核心对比逻辑 | 被数据噪音淹没 |
| **数字游民指南** | 完整的签证/VPN/时区板块 | 主要用户不是游民（反馈：30票） |

**这三个身份在争抢同一个首屏。**

### 2.2 功能膨胀的客观证据

来自 500 人模拟反馈：

- 生活压力指数：**0/500 人主动关注**，却占首屏最大卡片
- 购房年限：**30/500**，与月租（430票）占相同视觉面积
- 时薪：**20/500**，与年工时（160票）占相同面积
- 数字游民板块 14 个数据格：核心游牧数据仅 30-110 票
- 子指标（15个）：**0/500** 主动关注

这不是 "数据太多" 的问题——是 **信号-噪音比太低**的问题。

### 2.3 用户画像覆盖失衡

| 用户群 | 产品满足度 | 权重（潜在市场大小） |
|--------|-----------|---------------------|
| 跨国职场人 | ★★★★☆ | 极大（核心群体） |
| 数字游民 | ★★★★★ | 小（最被过度服务的群体） |
| 留学生 | ★★☆☆☆ | 大 |
| 有孩家庭 | ★★☆☆☆ | 大 |
| 退休人士 | ★★★☆☆ | 大 |

**投入最多精力的群体（游民）市场最小。投入最少的群体（留学/家庭）市场最大。**

## 3. 产品定位重塑

### 3.1 一句话定位

> **WhichCity 是全球移居决策引擎。**
>
> 用户选择职业和身份，我们用数据告诉他：去哪座城市能过上更好的生活。

不是城市百科、不是数字游民社区、不是数据 API 平台。

### 3.2 核心场景排序

| 优先级 | 场景 | 用户画像 | 行为模式 |
|--------|------|---------|---------|
| **P0** | 工作搬迁 / offer 对比 | 25-40 岁职场人，拿到 offer 或在考虑跨国跳槽 | 搜城市 → 看收入/成本/储蓄 → 对比 2-3 城市 |
| **P1** | 长期移民规划 | 25-45 岁技术移民/投资者 | 看排行 → 筛选条件 → 深入对比 |
| **P2** | 留学选城市 | 18-30 岁学生 | 搜城市 → 看成本/安全/语言 → 对比 |
| **P3** | 好奇研究 | 任何年龄，"XX城市怎么样？" | 社交媒体点进来 → 扫一眼 → 分享 |
| **降级** | 数字游民 | 已有完善竞品（NomadList） | 保留数据，降低视觉权重 |

### 3.3 与竞品的定位差异

| 竞品 | 定位 | WhichCity 的差异化 |
|------|------|-------------------|
| **Numbeo** | 生活成本数据库（UGC 众包） | WhichCity = 职业化×税后×决策导向，不是原始数据 |
| **NomadList** | 数字游民社区 + 数据 ($99/年) | WhichCity = 面向更广泛的移居人群，免费 |
| **Teleport** | 城市探索 + 生活质量评分 | WhichCity = 更深的财务计算（81国税制是杀手锏） |
| **Expatistan** | 生活成本对比 | WhichCity = 超越成本，涵盖安全/医疗/自由 |

**WhichCity 的核心竞争力是：职业×税制×多维度综合——在免费产品中独一无二。**

81 国税制引擎（含外派方案）是我见过的个人项目中最被低估的技术资产。这东西要商业化可以值六位数（美金）。目前它只是设置面板里一个 "net/gross/expatNet" 的 toggle。

## 4. 数据战略

### 4.1 数据层级重新划分

| 层级 | 数据 | 展示策略 |
|------|------|---------|
| **L1 核心** | 安全、月消费、月租、年收入（税后）、医疗、英语水平 | 首屏，大字，永远可见 |
| **L2 重要** | 年储蓄、气候、自由、语言、空气质量、工时、直飞 | 首屏下方，中等字号 |
| **L3 补充** | 税率、房价、假期、VPN、网速、日照、房价/m² | 紧凑展示 |
| **L4 深度** | 子指标(15个)、游牧签证详情、免签矩阵、时区重叠、购房年限、时薪 | 折叠/按需展开 |

### 4.2 强烈建议新增的数据（二阶段）

| 数据 | 理由 | 来源 | 工作量 |
|------|------|------|--------|
| **公共交通评分** | 100/500 需求，涵盖学生/家庭/老人 | Numbeo Traffic Index 或 Moovit | 中（154城手动/半自动） |
| **自然灾害风险** | 搬迁硬性排除项，如东京地震/迈阿密飓风 | World Risk Index（国家级可快速入） | 小 |
| **外国出生人口比例** | 代理"多元化/侨民友好"（220票需求） | 联合国/世行，国家级 | 小 |

### 4.3 可在三阶段考虑

| 数据 | 说明 |
|------|------|
| 国际学校数量 | 家庭用户核心，但数据获取难度中等 |
| 失业率 | 国家级数据易得，城市级更有价值但难 |
| 女性安全指数 | Georgetown WPS Index，一步可入 |

### 4.4 明确不做

| 数据 | 理由 |
|------|------|
| 夜生活/社交场景 | 不可量化 |
| 特定饮食友好度 | 碎片化 |
| 宗教设施/残障设施 | 标准化数据不存在 |
| 行业薪资天花板 | 26 职业已足够；P25/P75 分位数据难获取 |
| 共享办公密度 | 服务游民，不是核心用户 |

## 6. 职业增删建议

### 6.1 当前 26 职业评估

当前职业列表分析：

| 类别 | 职业 | 评估 |
|------|------|------|
| **科技** | 软件工程师、数据科学家、产品经理、UI/UX 设计师 | ✅ 核心用户群 |
| **专业服务** | 律师、医生、牙医、药剂师、会计师、建筑师 | ✅ 高搜索价值 |
| **管理** | 市场经理、销售经理、人力资源经理 | ✅ 合理 |
| **金融** | 财务分析师 | ⚠️ 仅1个，但够用 |
| **教育医疗** | 教师、护士、大学教授 | ✅ 大众职业 |
| **蓝领/服务** | 厨师、家政、公交司机、电工、机械工程师 | ⚠️ 见下 |
| **媒体创意** | 记者、平面设计师、摄影师 | ⚠️ 见下 |
| **政府** | 公务员 | ❌ 问题项 |

### 6.2 职业问题清单

**问题1：公务员数据不可靠**

公务员薪资在各国差异极大（日本公务员≈日本企业均薪；美国联邦公务员≈教师水平；中国公务员数据不透明），且不存在"国际公务员"这个流动场景——公务员不跨国搬家。

→ **建议删除**。替代方案：将其重命名为"政府行政人员(Government Administrator)"，使其适用于 NGO/国际组织等场景。

**问题2：家政服务人员薪资($800-$38,000)过于失真**

家政在发展中国家是非正规经济一部分，数据几乎不可能准确。且这个群体不会使用英文/日文版城市对比工具。

→ **建议删除**。

**问题3：摄影师和记者 — 自由职业化严重**

这两个职业在绝大多数城市不存在"标准薪资"。记者可能只有 4/500 人关注（报告中记者画像）。摄影师几乎完全按项目计费。

→ **建议保留记者（新闻自由是产品独特卖点之一），删除摄影师**。

**问题4：缺少远程工作者通用标签**

500人模拟中，数字游民/远程工作者是重要群体。但他们的收入不取决于目的地城市。

→ **不建议新增"远程工作者"职业**，因为其收入跟目的地城市无关。远程用户应选择自己的实际职业 + 调整薪资倍率。

### 6.3 建议最终清单

**删除 3 个**（26 → 23）：
1. ~~公务员~~ → 重命名为「政府/NGO行政」
2. ~~家政服务人员~~
3. ~~摄影师~~

**新增 0 个**。23 个职业已覆盖绝大多数有国际流动性的工作。更多职业 = 更多低质量数据。

**职业分组建议**（UI层面，不影响数据）：

```
💻 科技: 软件工程师 · 数据科学家 · 产品经理 · UI/UX设计师
💼 商业: 财务分析师 · 市场经理 · 销售经理 · 人力资源经理 · 会计师
⚖️ 专业: 律师 · 医生 · 牙医 · 药剂师 · 建筑师 · 大学教授
🏥 服务: 教师 · 护士
🔧 技术: 机械工程师 · 电工
🍳 其他: 厨师 · 记者 · 平面设计师 · 公交司机
```

### 7.3 税制引擎的产品化（关键建议）

**这是全产品最被低估的资产**。

81 国税制 + 外派方案 + 城市覆盖——这个能力在免费工具中独一无二。但目前它被藏在 NavBar 设置面板的一个 "gross/net/expatNet" toggle 里，用户几乎感知不到。

**建议**：在城市详情页创建一个「税务计算器」展开区域：


## 9. 市场策略

### 9.1 目标市场排序

| 排名 | 市场 | 特征 | 获取渠道 |

### 9.2 SEO 策略（最重要的增长引擎）

WhichCity 的自然搜索潜力巨大但完全未开发：

**长尾关键词矩阵**（每个都可以是一个排行发文章）：

```
"best cities for software engineers salary 2026"
"software engineer salary new york vs london after tax"
"cheapest safe cities to live in 2026"
"safest cities in the world ranking"
"highest paying cities for doctors"
"best cities for families safety healthcare"
"digital nomad visa countries 2026"
"tokyo vs singapore cost of living comparison"
```

**行动建议**：
1. 城市详情页的 meta description 优化为包含核心数据数字
2. 每个对比页的标题包含"vs"关键词
3. 考虑在排行页生成静态的职业×维度的长尾页面（例如 `/en/ranking/software-engineer/salary`）

### 9.3 病毒传播策略

最可能被分享的内容形式：

1. **排行榜截图** — "全球安全城市 Top 10"
2. **城市 VS 城市** — "纽约 vs 伦敦，谁赢了？"
3. **震惊类数据** — "在旧金山税后年收入 $118K，在苏黎世是 $147K"
4. **税务对比** — "同样 $100K 收入，新加坡实际到手比纽约多 $25K"

→ OG 图片应该针对这些场景优化，包含具体数字而不是通用设计。

## 10. 营收可能性

### 10.1 短期（0-6 月）：不追求营收

理由：
- 用户基数未知，过早变现会破坏增长
- 免费是对抗 NomadList ($99/年) 和 Teleport 的核心策略
- SEO 和自然增长需要时间

### 10.2 中期（6-18 月）：如果流量起来

| 模式 | 预估可行性 | 详情 |
|------|-----------|------|
| **AdSense / 展示广告** | ★★★★☆ | 最简单，工具类站点 RPM 约 $3-8/千次。10万月PV = $300-800/月 |
| **付费高级功能** | ★★★☆☆ | 导出 PDF 报告、更多对比城市、历史趋势 |
| **联盟营销** | ★★★☆☆ | 搬迁服务、国际保险、汇款服务的推荐链接 |
| **数据 API** | ★★☆☆☆ | B2B 需要稳定更新承诺，1 人维护成本高 |
| **赞助** | ★★☆☆☆ | 需要品牌知名度 |

**最推荐路径**：AdSense + 联盟营销（搬迁/保险/税务服务）。一个人可以运营，不增加产品复杂度。

### 10.3 税制引擎的独立变现潜力

如果将 81 国税制引擎包装为独立产品/Widget/API，有以下可能性：

| 形式 | 目标客户 | 价格区间 |
|------|---------|---------|
| 可嵌入税务计算器 Widget | HR SaaS / 招聘平台 | $500-2000/年 |
| Tax API (REST) | 跨国招聘公司 / EOR 平台 | $2000-10000/年 |
| 白标报告 | 咨询公司 / 搬迁服务商 | 按报告收费 $50-200/份 |

**这不是二阶段的事，但值得记住：税制引擎是潜在独立产品。**

## 13. 不做清单

**以下事项明确不做，无论看起来多诱人：**

| 不做 | 理由 |
|------|------|
| 用户账号/登录系统 | 1 人团队维护不起，且无必要 |
| 用户评论/UGC | 需要审核机制，引入法律风险 |
| 社区功能 | NomadList 已做，且需要运营资源 |
| 移动 App | Web 已够用，App 分散精力 |
| 更多语言（>4） | 翻译维护成本线性增长 |
| 实时数据更新 | 城市数据半年更新一次已足够 |
| AI 推荐引擎 | 简单排序就够用，不引入 AI 黑箱 |
| 夜生活/美食/文化评分 | 不可量化，违背产品边界 |
| 超过 160 座城市 | 数据质量 > 数量 |
| 超过 25 个职业 | 同上 |
| 教育模块（二阶段） | 数据获取工作量大，三阶段考虑 |
| 付费功能（二阶段） | 先有用户再谈变现 |

## 14. 总结：一句话路径

**WhichCity 的路径不是"做更多"，而是"让已有的东西被看见"。**

你已经有了：
- 154 座城市的 50+ 维度数据
- 81 国税制引擎（免费世界唯一）
- 4 语言国际化
- 完整的 SEO 基础设施

你缺的是：
- 一个清晰的信息层级，让用户 3 秒内知道「这是给我用的」
- 把税制引擎从设置面板里解放出来，变成用户看得见的杀手功能
- 让安全/消费/收入这三个全民共识数据占据最显眼的位置
- 把游牧板块塞回折叠区，把英语水平拉出来

二阶段的核心任务可以用一句话概括：

> **把噪音压下去，把信号放出来。**

做完这件事，WhichCity 就能从一个工程师的个人项目，变成一个有独立产品认知的工具——一个每个考虑跨城市搬迁的人都会收藏的工具。
