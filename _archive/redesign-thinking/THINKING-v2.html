WhichCity — 第二轮思考

回应你的三个问题。

一、关于结论卡片

你提了两个精准的问题：

  1. 它和下方数据重复 → 浪费空间 + "凑数"感
  2. AI 味太重 → "玩具"感

问题的根源

结论卡片试图做的是"用一句话总结这个城市"。但这在根本上就有矛盾：

一个城市不能被一句话总结。
"安全 A，月存 ¥8 万"——这不是结论，这只是把两个数字串成一句话。
用户扫一眼三个大数字 + 四根横条就知道的事情，不需要一段话来"翻译"。

三个方案

方案 1：删掉卡片，直接进入数据

城市名 + 简介 → 直接接三个大数字。没有中间层。最干净。

风险：首屏太"冷"，没有情感抓手。用户落地后看到城市名+一堆数字，感受像数据库。

方案 2：保留"品牌区"，但换内容

你说得对，"WhichCity 评估"这个概念有品牌价值。问题不是品牌区本身，是里面装的内容。

如果品牌区不装"AI总结段落"，装什么？

装标签。只装标签。

标签是结论卡片里唯一不和下方重复的东西。
"极低犯罪"、"英语弱"、"高税负"——这些是对数据的定性判断，用户扫一眼就获得"这城市大概是什么性格"的感觉，但不会在下面的数字区块里逐字重复。

而且标签天然适合扫描，不需要"阅读"。

视觉上：不用渐变背景、不用大边框。就是一行小标签，紧跟在城市简介下面，几乎不占空间。不需要"卡片"形态——它只是城市简介的一部分。

方案 3：把"安全评级字母"提到城市名旁边

不用单独的卡片区域。直接在城市名旁边放一个字母徽章：

🇯🇵 东京 A

这样安全评级在 0.5 秒内就被注意到，不需要任何额外的卡片空间。

我的建议

方案 2 + 3 组合：评级字母贴在城市名旁边，标签行跟在简介下面。没有独立的"结论卡片"。

安全警告城市（基辅、拉各斯）是特殊情况 —— 这些城市需要一个红色警告条（因为这是关乎人身安全的信息），但这不是"结论卡片"，这是"安全警告"。它只在 safetyWarning 存在时出现。

二、城市详情页的完整重设计

2.1 数据重新分组

现有分组是按"数据类型"分的：钱 / 安全 / 生活。这是程序员的分类方式，不是用户的思考方式。

用户的思考是按决策因素来的：

用户在想需要看到模块名
"活得起吗？" 收入、支出、结余、租金、购房年限 💰 经济
"安全吗？社会靠谱吗？" 安全指数、治理、腐败、政治稳定 🛡️ 安全与治理
"健康吗？舒服吗？" 医疗水平、AQI、气候、工时、假期 🌤️ 健康与生活
"能融入吗？方便吗？" 语言、英语水平、直飞、签证、VPN、时区 🔗 语言与连接

关键变化：

  • "数字游民"被拆散了 — 英语水平、VPN、签证信息融入"语言与连接"模块。不再有一个单独的"游民"区块（大部分用户不认为自己是"数字游民"，但他们关心签证和语言）
  • 医疗从"安全"分出来 — 医疗和 AQI、气候放在一起，因为用户想的是"在这个城市生活，我的身体会怎么样"
  • 治理和安全合并 — 腐败、政治稳定、法治——这些都在回答"这个社会值得信赖吗"

三、关键设计决策逐条说明

3.1 导航栏 — 职业选择内联

用户最关心的"设置"只有一个：职业。它直接影响所有收入数据。

所以职业选择器不应该藏在设置面板里。它直接放在导航栏上，显示当前职业名。点击下拉切换。用户零学习成本，看到就懂。

其他设置（语言、货币、主题）藏在 ⚙ 里。这些都是"设一次就忘"的东西。

3.2 颜色策略 — 克制

颜色使用场景判断标准
绿色明确好的数值安全 ≥70, AQI ≤50, 存钱率 ≥30%
琥珀需要注意月支出（提醒这是"花掉的钱"）、英语 B/C、AQI 51-100
红色仅安全警告safetyWarning 存在时
蓝色中性偏正面安全 50-69、自由指数
黑色默认所有无需判断的数值

原则：大部分数值是黑色的。 只有需要用户"注意"或"安心"时才上色。如果一个页面上 70% 的数字都有颜色，那颜色就失去了信号价值。

3.3 西语/长文本策略

问题对比：

中文西语长度比
安全与社会保障Seguridad y Protección Social1:4
税后月收入Ingreso Mensual Neto1:3
月结余Excedente Mensual1:5

解法：

  • 标签文字用最短形式 — "月入"而非"税后月收入"，"Net"而非"Net Monthly Income"，"Excedente"而非"Excedente Mensual"
  • 数字标签在数字下方，不在旁边 — 垂直排列永远比水平排列对长文本友好
  • section 标题不含长描述 — 用 emoji + 2-3 个短词（"💰 经济"、"💰 Economía"）
  • 永远用 flex-wrap — 标签行超长时自动换行，不截断

3.4 数字游民模块 → 拆散重组

"数字游民"这个概念本身是问题。

大部分用户不自我认同为"数字游民"。但他们关心语言、签证、VPN。
把这些信息放在一个叫"数字游民"的折叠区里，等于让多数用户跳过了他们需要的信息。

拆散后融入"🔗 语言与连接"模块：语言、英语水平、直飞、VPN、签证类型、时区重叠。
所有人都能看到，不需要"我是游民"这个心智前提。

3.5 Insight 文案 — 保留但精简

Insight box（浅色背景的一句话解读）保留，因为它做的事情和数字不同：它回答"so what"。

"医疗指数 78"——so what？
"每千人 2.7 名医生，覆盖指数 83/100，寿命 84.5 岁"——哦，这意味着医疗很好。

但每个模块最多一条 insight。不做多行多段。

3.6 折叠区策略

默认折叠理由
税务明细90% 用户不看。只有真想移的人才关心累进税率
安全/治理子指标综合分 + 横条已经够用。凶杀率、WGI 具体值是专业数据
月度气候图年均温和气候类型已经给出概况。月度数据是深入补充

不折叠的东西：经济三数字、安全横条、生活网格、语言与连接、相似城市 — 这些是决策核心信息，始终可见。

3.7 页面间视觉一致性

三个页面用同一套 CSS 变量、同一个 NavBar、同一种间距节奏。

页面配色风格
首页（搜索）几乎无色。白底 + 搜索框 + 灰色文字
排行列表无色。数字排行，只有 top 3 的排名数字是绿色
城市页少量语义色。标签、bar fill、个别数字。白底主导

三个页面看起来像同一款产品的三个视图，而不是三个不同风格的页面。

四、总结

核心改动清单：
  1. 结论卡片 → 删除。评级字母贴在城市名旁、标签行跟在简介后。安全警告是单独的红色条（仅特殊城市）。
  2. 数据分组 → 从"钱/安全/生活"改为"经济/安全与治理/健康与生活/语言与连接"。
  3. 数字游民 → 拆散融入"语言与连接"模块。
  4. 安全区块 → 医疗分离出去，安全和治理合并。横条从 4 根变 3 根（治安、治理、自由）。
  5. 导航 → 职业选择器内联在导航栏，其他设置藏在 ⚙。
  6. 配色 → 克制。大部分数字是黑色。只在明确好/坏时上色。
  7. 西语兼容 → 标签用最短形式，垂直排列，flex-wrap。
  8. 三个页面 → 同一套 CSS 变量，统一视觉语言。
