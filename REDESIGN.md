## 产品定位

**WhichCity 是全球移居决策引擎。**

用户选择职业和身份，我们用数据告诉他：去哪座城市能过上更好的生活。

不是城市百科、不是数字游民社区、不是数据 API 平台。

**核心竞争力**：职业 × 81 国税制 × 多维度综合 —— 在免费产品中独一无二。

---

## 第一部分：硬约束

以下条目是工程和产品的底线，任何改动都不得违反。

### 1. URL 兼容
- 现有 URL 结构 `/[locale]/city/[slug]`、`/[locale]/ranking`、`/[locale]/compare/[pair]` 不变
- 老链接不会 404

### 2. 数据完整性
- cities.json 的字段**只增不删**
- 前端可以选择不展示某些字段，但不得从数据源移除

### 3. SEO 基础设施
- OG images、sitemap、JSON-LD、metadata 保留
- 城市详情页的核心数据（收入、安全、消费）保持默认可见

### 4. SSR/Hydration
- `mounted` 机制保持不变
- 依赖 localStorage 的内容必须在客户端渲染

## 第二部分：Phase 2 设计原则

### 信号 > 噪音
当前问题不是"数据太多"，而是信号-噪音比太低。30+ 数据以相同视觉权重平铺。改进方向是**建立层级**：
- 重要的数字更大、更醒目
- 次要的数字更小、更安静
- 补充信息折叠

### 三秒规则
用户在任何页面，3 秒内应能回答"我在看什么"。

### 数据层级

| 层级 | 数据 | 展示策略 |
|------|------|---------|
| **L1 核心** | 安全、月消费、月租、年收入（税后）、医疗、英语水平 | 首屏，大字，永远可见 |
| **L2 重要** | 年储蓄、气候、自由、语言、空气质量、工时、直飞 | 首屏下方，中等字号 |
| **L3 补充** | 税率、房价、假期、VPN、网速、日照 | 紧凑展示 |
| **L4 深度** | 子指标(15个)、游牧签证详情、免签矩阵、时区重叠、购房年限、时薪 | 折叠/按需展开 |

### 展示层做减法的方式（按优先级）
1. 缩小视觉权重（字号、颜色、位置）
2. 收纳到面板/下拉
3. 折叠（默认隐藏，点击展开）
4. 移至其他页面
5. 删除（最后手段，需要明确理由）

## 第三部分：Phase 2 具体方案

### 城市详情页

**核心变化**：
- 安全指数提至首屏最显眼位置（500/500 关注）
- 英语水平从游牧板块移至 Hero 区（320/500 关注）
- 月消费排在年收入前面（440 > 370 票）
- 综合指数子指标**默认折叠**（0/500 主动关注）
- 游牧板块改为**可折叠**（核心仅 30-110 票）
- VPN 限制提至环境组（110 票）
- 税制引擎可视化：创建展开式「税务计算器」区域

**降级**：
- 生活压力指数：首屏大卡 → 排行保留 + 详情页折叠
- 购房年限：Row 1 平级 → 房价卡内小字
- 时薪：Row 2 平级 → 工时卡内小字
- 月度气候图：移至底部

### 排行榜

- 删除 Multi 复合排序模式
- 新增「税后收入」Tab
- 时薪合并入工时 Tab、购房年限合并入房价 Tab
- 生活压力移至末尾

### 功能删除

| 功能 | 理由 |
|------|------|
| Multi 复合排序 | 门槛极高，0 产品独特性 |
| LINE 分享 | 海外无用户 |
| Reddit 分享 | 中/日/西语用户无意义 |

### 职业调整（26→23）

| 操作 | 职业 | 理由 |
|------|------|------|
| 删除 | 家政服务人员 | 数据不可靠，用户不匹配 |
| 删除 | 摄影师 | 自由职业无标准薪资 |
| 重命名 | 公务员→政府/NGO行政 | 公务员不跨国流动 |

### 城市调整

**隐藏展示**（数据保留在 JSON）：帕岸岛、苏梅岛、暹粒、班斯科、库斯科、马拉喀什

**新增候选**：阿克拉（加纳）、利雅得（沙特）

### 数据新增

| 数据 | 来源 | 优先级 |
|------|------|--------|
| 公共交通评分 | Numbeo Traffic Index | P1 |
| 自然灾害风险 | World Risk Index | P1 |
| 外国出生人口% | 联合国/世行 | P2 |
